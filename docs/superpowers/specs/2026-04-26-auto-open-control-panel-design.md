# 自动打开控制面板设计

## 目标

游戏启动且插件正常运行后，在本机 HTTP 控制面板成功启动时，按配置自动调用系统默认浏览器打开真实控制面板地址。该功能默认开启，可在插件设置中关闭。

## 设计

- 在 `RuntimeConfig` 中增加 `AutoOpenControlPanel`，默认值为 `true`。
- 在 `UpdateConfigRequest`、`ControlPanelState` 和 `ControlPanelService` 中贯通该字段，使其能通过现有 `settings.json` 持久化。
- 插件启动流程保持现有顺序：先启动 `LocalHttpServer` 并获得实际 URL，再根据 `AutoOpenControlPanel` 调用系统默认浏览器。
- 浏览器打开动作只在本次游戏进程启动时执行一次，不参与运行时轮询；如果 HTTP 服务启动失败，保持现有失败日志，不尝试打开浏览器。
- 控制面板设置页增加一个开关，中文显示为“启动后自动打开控制面板”。

## 错误处理

浏览器打开失败不应影响翻译插件运行。失败时只写入警告日志，HTTP 面板、翻译队列和捕获流程继续工作。

## 测试

- 单元测试覆盖默认值为开启。
- 单元测试覆盖设置保存后重新加载仍为关闭。
- 单元测试覆盖 `GetState` 返回该字段，供前端开关使用。
- 构建验证覆盖插件项目编译通过。
